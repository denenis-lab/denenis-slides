Процесс

Два подхода

Skill Creator

1. Understand — примеры
2. Plan — ресурсы
3. Init — шаблон
4. Edit — написать
5. Package — упаковать
6. Iterate — улучшать

Writing Skills

1. RED — тест без skill
2. Задокументировать провалы
3. GREEN — написать skill
4. Тест со skill — pass
5. REFACTOR — закрыть лазейки
6. Ре‑тест до bulletproof

Вместе

Комбинированный процесс

Собрать примеры, спланировать → Skill Creator
Baseline‑тест без skill → Writing Skills
init_skill.py + написать → Skill Creator
Pressure‑тест, закрыть лазейки → Writing Skills
Валидация + упаковка в .skill → Skill Creator
9 / 12

Когда какой

Выбор по задаче

Только Creator

Простой reference‑skill. API‑доки, шаблоны, ассеты. Нечего тестировать

Только Writing

Discipline‑skill для себя. Не нужна дистрибуция

Оба

Skill с правилами + публикация. Нужен и формат, и надёжность

Ни один

Правила одного проекта → CLAUDE.md. Не создавай skill

10 / 12

Вердикт

Оставить оба.

Skill Creator — стандарт формата и инструменты сборки. Writing Skills — методология качества и тестирования. Не дублируют, а дополняют друг друга.

Вызывать отдельно не нужно — Claude сам выберет нужный по контексту задачи.
